Summray

The Wizio API allows you to display 360 content on your website dynamically.

Utilizing the Wizio API

The API utilizes iframes to display Wizio 360 tours. A request must be sent to Wizio's API with an API token, an address and a unit number to receive the dynamically generated link to the tour. Once the link to the tour is received it should be placed in an iframe src attribute.

Request

The request made to the Wizio API must be a GET request sent to https://api.wizio.co/tour/:apitoken/:address/:unitnumber

  • apitoken: Your API token provided by Wizio.
  • address: Address of the tour you're looking to pull.
  • unitnumber: Unit number of the tour you're looking to pull.

Response

The response from the Wizio API follows the following formats:


        {
          status: 'success',
          success: 1,
        }

Welcome to the Wizio API Guide!


What is the Wizio API?

With the Wizio API you'll be able to request and put virtual reality images right onto your own listing pages with ease. The first thing you will need is an API key/access token.

How do I get an API key?

If you don't already have an API key/access token, please request more information on attaining one by clicking the button below and filling out the super brief form.




How do I utilize the Wizio API?

A developer, most likely your current website manager/developer, will use your unique API key to send a request to our web application for the apartment address and unit number you are interested in having virtual reality for. Upon receiving a request, our web application provides an iframe source, an adjustable window that can be embedded into a web page for viewing the virtual reality content for the specific address requested. The API works by placing a specific URL in an iframe src attribute. This URL is dynamically generated by the Wizio API server upon a proper and validated request. The request must be a GET request to api.wizio.co.
Here is an example request - A full request will look like the following:

  • api.wizio.co/api/vr/listing/46a17cc1-3310-11e6-a0b2-0800274a1eca/175_amory_st_jamaica_plain_ma/2
  • api.wizio.co/api/vr/listing/{apikey}/{apartmentAddress}/{unitNumber}

The request can be broken up in the following way, in parameter order
  • /api/vr/listing/ --- this is a static part of the address. It will always be the same.
  • 6a17cc1-3310-11e6-a0b2-0800274a1eca/ --- This is an example API key that you will receive from us.
  • 175_amory_st_jamaica_plain_ma--- This is the apartment address. The apartment address needs
    • Any spaces to be replaced with '_' (underscores)
    • A nearly complete address. The address does not need country, but it should have at least a street number, a street, a city/town and a state. A zip code is preferable.



The response from the GET request will look like the following:
  • http://beta.wizio.co/listing/vr/6f6e7849-a6d5-46be-96a0-6aef59c5ce5d/08518c8d-327f-11e6-a0b2-0800274a1eca
This is the URL that must be placed into the src attribute of your iframe. From here the iframe will take over loading the rest.

How do I know my key works?

Wizio has test data that you may use for testing to make sure the API is working. Below are the following addresses that test data exists for. To test, you'll want to make the request as noted above in the documentation with some of these hard coded addresses. The request from each of these responses will be a URL. You need to load this URL into the iframe's src attribute on your page.

  • 101 Monmouth Street boston ma unit number 1
    • api.wizio.co/api/vr/listing/APIKEY/101_monmouth_st_boston_ma/1
